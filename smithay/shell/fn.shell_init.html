Create new xdg_shell and wl_shell globals.

The globals are directly registered into the eventloop, and this function returns a StateToken<_> which you'll need access the list of shell surfaces created by your clients.

It also returns the two global handles, in case you whish to remove these globals from the event loop in the future.
